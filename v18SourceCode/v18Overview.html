Overview

 

Welcome to the v18 alpha web design standards site! This new design system is created with Adaptive Web Design (AWD) enabling websites to be viewed on desktops, tablets and mobile devices all with one set of code. The code is built in HTML5, CSS3 and jQuery with numerous advantages and optimized for mobile devices. Everyone is welcome to view this site, but adoption of v18 alpha is limited. Only those who have been accepted into the alpha program can test and deploy a v18 site. To learn more and get started, review the information below.

Adaptive Web Design

Adaptive Web Design (AWD) is a design method that focuses on the user. Using predefined layout sizes, based on device screen size along with CSS and JavaScript, the AWD approach adapts the user interface to each device.

It transforms web pages to be viewable on smaller screen sizes (for example, tablets and cell phones). Go ahead – make your browser window smaller... see how this page changes to fit smaller screen devices.

New designs

Two of today's prominent design patterns are now available in v18 alpha. The first is Full-bleed images which can dramatically emphasize your message by presenting high-resolution images that expand or contract to the browser window's width,

resulting in an impressive edge-to-edge look. The second is Cards, a design pattern that combines video, categorization, text and imagery into a single container. Cards also provide a clean, visual option to standard text paragraphs or containers.

HTML5

HTML5 delivers rich web content including support for video and audio without additional plugins. It improves Search Engine Optimization (SEO) by using cleaner markup enabling search engines to index a

page faster and more accurately. It also improves accessibility by using specific sectioning elements and assistive technology that can help the user more easily navigate the page.

CSS3

What makes CSS3 simple is that it is split into logical and user-friendly modules. Within each of these modules is a set of customizable functions, which allows for greater flexibility in development. CSS3 enables you to create beautiful user

experiences by having animations, transitions, gradients, web fonts, multiple backgrounds, CSS columns, 3D transforms and most importantly, media queries which are used to create UIs for different size screens.

jQuery vs. Dojo

When v17 was introduced four years ago, the Dojo JavaScript library was deemed the corporate strategic direction. Recently, that direction has been changed and sites/apps can use any approved technology that works best for its target environment.

v18 is built with the jQuery JavaScript library which is lightweight, loads fast and is ideal for mobile devices. You can use Dojo or other approved JavaScript libraries with v18, but that will require loading more files which can affect page load performance.

Release plans

Similar to a product, v18 will be released in phases and there will be several releases of v18 (alpha, beta, etc.) up until we deliver a general availability (GA) release in 2015. The term, "alpha" means an early release or the first release of code.

It does not contain all the planned features, capabilities and will initially look very much like v17. It is intended for early adopters looking to deploy smaller sites and collaborate with us on testing new features and designs while accepting some risks.

Adoption routes

There are several paths to adopting v18. If you are a new site or small site, then the alpha can be a good place to start. If you're a medium size site or site with less traffic, then the beta may be right for you. If you are a large, complex or highly visited site, then the GA release is the best choice. v17 sites

should consider the beta or GA versions while v17e sites can go with alpha or beta.

Comparing v17e

In 4Q 2013, the v17e pilot project was started to explore new designs and a mobile UI. For some sites, you may want to perform the upgrade to v17e to get the mobile UI, remain on the Dojo code base and upgrade to v18 beta in 2015.

Design Language

The IBM Design Language (DL) is developed by the IBM Design team and is intended for IBM products. The DL is a set of living guidelines that is more instructive than prescriptive. This effort is relatively new and crafted intentionally to evolve

through feedback from product teams and users. The v18 team is collaborating with the IBM Design team to ensure our web experiences are consistent between websites and products.

Boarding process

Depending on the size of your site, the number of locales supported, the amount of traffic your site receives and other criteria, your site may be selected as a v18 alpha early adopter. Adopters must accept some risk for outages, frequent code

changes, partner in testing, no developing custom code and other conditions. For the alpha, adoption is limited, but the beta version will be open to all. We will select new alpha adopters each month until early 2015.

CMAD exception required

In order to use the v18 alpha design system, you are required to obtain a CMAD exception. All v18 alpha CMADs are granted for six months and will enable your site to be published to production. Next year, v18 beta will be available and all v18 alpha sites

will be expected to upgrade before their CMADs expire. CMAD submissions will be reviewed against certain acceptance criteria to be a candidate for v18 alpha early adoption.

What is v18 alpha?

The alpha version leverages the agile and iterative software development process. There are new code releases twice a month. Code development is done in collaboration with early adopters who test and report issues from various hosting

environments. Visit the v18 development site for current information on supported browsers and browser versions, which designs and features are currently in the alpha and what is coming soon.

Getting started

To start the process, review the Getting started page, submit a CMAD request and if that submission is accepted, your site will be a v18 alpha early adopter! The v18 alpha code base will contain a URL list that

restricts usage to those sites who had their CMAD accepted. Everyone is welcome to view this web design standards site, but only sites who have been accepted can test and deploy into production.
